using namespace std 이제는 쓰지맙시다.

Don't Use using namespace std.

C++을 사용하는 많은 분들이 std::cout, std::cin, std::endl을 전부 작성하기가 귀찮아서 using namespace std;을 통해 생략하곤 합니다. 하지만, 이 방법은 오히려 코드가 점차 늘어날수록 지양되어야 하는 방법입니다. 왜냐하면, namespace을 독립시키고자 사용했던 std라는 namespace가 불려오면서 다른 namespace와 충돌이 일어나면 원하는 방식대로 프로그램이 동작하지 않을 수도 있기 때문입니다. 그렇다면, 항상 이 방식 대신에 std::cout처럼 std::사용을 고수해야만 할까요?

꼭 그런 것만은 아닙니다. 우리에게는 Macro Function이라는 아주 좋은 수단이 있으니까요. Marco를 사용하면, 코드를 짧게 만들면서도 namespace의 충돌문제를 해결할 수 있는 좋은 방법입니다. Macro Function을 이미 잘 알고 계시다면, 가장 아래쪽으로 그렇지 않다면 Macro Function 부분을 먼저 읽어주세요.

Most C++ coders feel inconvenient to type the std::cout, std::cin and std::endl all and skip the part of std:: with using namespace std;. However, the longer codes the program has, the more this method should be unrecommended. It's because the isolated and independent namespaces can be conflicted together so that the program might not work as you have intended. Then, should you write whole words everywhen making programs?

No, you can handle this issue with a very good means called Macro function. If using Macro functions, then you can make the codes shorter and resolve the namespaces-conflict issue. If you already know about the Macro function, then please read the section of 2. Application and if not, please read the section of 1. Macro Function.

1. 매크로 함수 (Macro Function)

1. Macro Function

Macro 함수의 문법은 매우 쉽습니다. Macro와 관련된 모든 문법은 C/C++에서 #define A B의 형식으로 사용하게 됩니다. 이 #define의 의미를 한 단어로 생각하시면 됩니다. 바로 '대체한다'입니다. 여러분들이 Macro Constant 또는 Macro Function을 사용할 때에는 항상 'A를 B로 대체한다'의 의미를 갖고 가시면 절대 헷갈릴 일이 없습니다.

A syntax for the macro function is very easy. In C/C++, all syntaxes for the macro has a format of #define A B. You must consider the meaning of #define, which means 'replace'. Whenever using macro constants or macro functions, you should keep it in mind 'to replace A into B'. Then, it will never confuse you. 

#include <iostream>

#define PI 3.141592 // Macro Constant
#define CUBIC(x) x*x*x // Macro Function

int main(){
  cout << PI << endl; // Replace PI (A) into 3.141592 (B).
  cout << CUBIC(3) << endl; // Replace CUBIC(3) (A) into 3*3*3 (B).
  return 0;
}

2. 솔루션 (Soltuion)

2. Application (Soltuion)

Macro의 기능에서 거의 눈치채셨듯이 Macro를 사용해서 cout이 std::cout으로 대체되는 형식으로 구성하면 우리가 원하는 결과를 얻을 수 있습니다. 이렇게 하면, 짧은 형태의 cout은 그대로 사용할 수 있으면서도 namespace의 충돌은 막을 수 있습니다. 아래의 예제 코드를 통해 실제 사용을 한 번 들여다 봅시다. 하지만, 이러한 Macro의 단점은 너무 빈번하게 사용한다면 Compile 시간을 증가시킨다는 단점이 있습니다.

Using a functionality of macro, you can make cout replaced into std::cout. Then, meanwhile using a short format of cout, you can prevent namespace from conflicting each other namespaces. Let's take a look at the below example. However, there is a weakness that increase a compile time if you use macro too frequently. 

#include <iostream>

#define cout std::cout
#define endl std::endl

int main(){
  cout << "Hello, World" << endl;
  return 0;
}
