MacOS C/C++ 세팅하기

English version is not supported.

0. Preliminary Knowledge

그냥 강하게 말하겠습니다. 이 문서 설명 읽고도 세팅 못하면 현실적으로 그냥 컴퓨터 접어야 하는 수준임. 일단 재능없으신 겁니다. MacOS에서 C/C++을 실행하는 가장 간단한 방법은 Visual Studio(비주얼 스튜디오, 이하 VS)를 설치해서 사용하는 방법이다. 하지만, VS에서 C/C++을 사용하기에는 게임 개발과 같은 협업이 굉장히 중요하고, VS의 부가 기능들을 필수적으로 사용하는 경우를 제외하고는 대부분의 경우(교육/학습/에뮬레이터)에는 GNU에서 개발된 GCC/G++를 사용하는 것이 좋다. 우리는 MacOS에서의 사용을 위해 Terminal(터미널)이라는 것을 알아야 한다. Terminal은 윈도우의 cmd(혹은 powershell)과 같은 역할을 하는데 명령어를 입력하면 명령어에 따라 프로세스가 동작한다. 터미널은 Lunchpad > 기타 > 터미널에서 실행시킬 수 있다. 또는 ⌘ + space로 Spotlight(스포트라이트)를 실행시켜 터미널을 검색해 실행시킬 수 있다. 실행 후, 터미널 아이콘이 화면의 사이드바에 뜨는데 이 아이콘을 가장 왼쪽 섹션으로 옮기면 터미널이 사이드바에 고정될 것이다. 터미널을 종료시켜도 사이드바에서 아이콘이 사라지지 않는다. 추가적으로 기본 터미널 보다 다른 터미널을 선호한다면, iTerm을 선택적으로 다운로드 받을 수도 있다. 경로에 대한 설명은 추후 설명하도록 하겠다...!

1. Prerequisites

사전 요구 프로그램 설치하기에 앞서, 본인의 프로그램 상태를 확인해보자. 

g++ --version

위의 명령어 결과가 아래와 같다면, 3번으로 가자. 

g++ (Homebrew GCC 13.2.0) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.
This is free software; see the souce... (생략)

MacOS에서 여러 프레임워크, 어플리케이션 등을 터미널을 통해 다운로드 받기 위해서는 Homebrew라고 하는 프로그램이 필요하다. 아래의 명령어를 터미널에 입력해 설치해주자. 

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

중간에 Password 치라고 나올텐데, 당황하지 말고 본인의 MacOS 비밀번호를 타이핑하고 엔터를 눌러주자. 이렇게 하면, 사전 설치 프로그램 설치는 완료했다. 하지만, 프로그램이 잘 동작하는지 먼저 확인해주자. 

brew

위의 명령어를 입력했을때 아래의 결과가 뜨면 안 된다.

zsh: command not found: brew

결과가 나왔다면, 6번으로 경로 재설정을 하고 돌아와야 한다. Visual Studio Code 같은 경우는 직접 사이트 들어가서 설치하자. 또한, xcode commandline도 Terminal을 설치해주어야 한다. 이 작업을 전에 한 적이 있다면(다른 프로그램 설치를 위해서라도), 생략하도록 한다.

xcode-select --install

2. Installation

아래의 명령어는 Homebrew를 통해 gcc/g++ 설치를 진행해준다.

brew install gcc

다시 한 번 설치 상태를 확인해보자.

g++ --version

위의 명령어 결과가 아래와 같지 않을 것이다.

g++ (Homebrew GCC 13.2.0) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.
This is free software; see the source... (생략)

그렇다면, 아래의 명령어를 실행해본다. [버전]의 의미는 gcc/g++ 버전 중 가장 큰 범주의 버전을 의미한다. 예를 들어 GCC 13.2.0이라면 13을 의미한다. 이 버전은 brew install gcc의 설치 히스토리(터미널 상에 떠있을 것임)를 보면 된다. 

g++-[버전] --version

위의 명령어 결과가 아래와 같을 것이다. 

g++ (Homebrew GCC 13.2.0) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.
This is free software; see the source... (생략)

그렇다면 잘 설치가 된 것이다.

3. Symbolic Link

이 과정은 귀찮은 g++-[버전] 작성을 덜어주는 작업이다. 터미널이든 iTerm이든 기본적으로 명령어 수행을 위해 통과 및 검색하는 경로는 /opt/homebrew/bin이다(추가로 몇 개 더 있음). 하지만, 본인이 이 경로에 gcc/g++ 명령어를 추가하고 싶지 않을 수도 있다. 다른 경로에 추가하고 싶다면, 더 공부하고 추가하도록 하자. 일단, 이 글을 읽는다는 것자체가 컴퓨터에 대해 잘 모른다는 의미이니 이상한 쌩쇼하지 말고 이 메뉴얼을 잘 따르자. 아래의 명령어를 실행해보자. 

cd /opt/homebrew/bin
ls

실행 결과 이상한 것들이 출력이 엄청나게 될 것이다. 여기서 스크롤해가며 g++-[버전]이 있는지 확인해보자. 있다면, 아래의 명령어를 실행해보자. 없다면 유튜브 채널에 댓글 ㄱㄱ. 

./g++-[버전] --version

명령어의 결과가 아래와 같아야 한다. 같지 않다면 유튜브 채널 댓글 ㄱㄱ. 

g++ (Homebrew GCC 13.2.0) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.
This is free software; see the source... (생략)

다음으로 아래의 명령어를 실행한다. 

sudo ln -s g++-[버전] g++

비밀번호는 알아서 입력하면 된다. 여기까지 했으면 터미널 창을 전부 닫아주고 MacOS도 로그아웃 후에 다시 로그인해준다. 본인이 로그아웃이 뭔지 모르겠다면, 그냥 MacOS 전원을 껐다가 다시 켜주면 된다.

4. Test

다시 터미널을 켜준다. 다시 아래의 명령어를 실행해보면, 이제는 Homebrew GCC가 잘 뜰 것이다. 

g++ --version
g++ (Homebrew GCC 13.2.0) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.
This is free software; see the source... (생략)

C++ 코드를 작성하고 잘 동작하는지 보자. 

code /Users/[본인이 코드를 작성할 경로]

Visual Studio Code에서 C++ 파일을 main.cpp이름으로 하나 만든다. 

// test/directory/main.cpp
#include <iostream>

int main(){
  std::cout << "Hello, World!\n";

  return 0;
}

명령어를 실행시켜서 잘 동작하는지 보자. 

g++ -o runfile main.cpp -lm -ld_classic
./runfile

Hello, World!가 잘 출력되면 성공이다. 끝!

5. Addition of Compilation Option (추가사항!!! 필독!!!)

저의 유튜브 영상을 본다면, 아래와 같은 명령어로 컴파일을 실행했을 것입니다. 

g++ -o runfile main.cpp -lm

여기서 -lm은 수학 라이브러리인데, 코딩을 하면서 수학을 안 쓰는 일은 거의 없기 때문에 항상 적으면서 컴파일해주는 습관을 들이면 좋습니다. 하지만 이제부터는 Compile 명령어를 아래와 같이 다르게 작성해야 합니다. 

g++ -o runfile main.cpp -lm -ld_classic

그리고 -ld_classic은 MacOS 버전과 GNU G++ 사이에서 호환성 문제로 인해 발생하는 에러를 없애줍니다. 이 옵션이 없다면, 계속해서 Compile Error가 납니다. 이와 관련해서 Apple측에서 조만간 해결하겠다고는 하는데 언제 수정이 될지 잘 모르겠습니다. (얘네들 해결하면 알려드림 ㅇㅇ)

6. Profile Setting

이 섹션을 읽는다면, brew 명령어가 동작하지 않기 때문일 것입니다. 이번에도 다시 한 번 명령어를 실행해 결과가 어떤지 확인해보겠습니다. 

cd
cat .zshrc

여기서 결과가 다음과 같이 나오는지 확인해 봐야 합니다. 아마 결과가 아예 없거나 아래의 결과가 존재하지 않을 겁니다. 아래의 결과와 완전히 똑같아야 한다는 의미는 아니며 아래의 결과가 위의 명령어 실행 결과에 '포함'되어 있어야 합니다. 

export PATH="/opt/homebrew/bin:$PATH"

있다면, 아래의 명령어를 실행해주신 다음에 brew 명령어로 잘 동작하는지 확인해주시면 됩니다(이 이후에도 정상 동작하지 않는다면, 결과가 없던 것처럼 생각하고 이어나가 주시기 바랍니다). 만약 없다면, 아래의 명령어를 실행하지 마시고 ▶표시가 있는 줄로 가서 읽으시면 됩니다. 

source .zshrc

▶ 여러분들은 타이핑을 하셔야 합니다. 아래의 명령어를 실행하면 이상한 모양으로 터미널이 바뀔 것입니다. 

nano .zshrc

여기서 키보드 i를 누르고 export PATH="/opt/homebrew/bin:$PATH"를 똑같이 입력하시면 됩니다. 그리고 ESC를 누른 후, :, w, q, Enter를 순서대로 누르시면 됩니다. 이때 : 누를 때는 shift도 눌러주셔야 합니다!!! 그러면 이제 다시 기존의 화면이 나올 겁니다. 그곳에서 아래의 명령어를 실행해주시면 됩니다. 

source .zshrc

그리고 brew 명령어의 동작 여부를 살펴보면 정상 동작할 것입니다.
